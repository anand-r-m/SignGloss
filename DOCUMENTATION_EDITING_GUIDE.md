# SignGloss — Editing Guide

A beginner-friendly, practical guide to safely modifying anything in this project.

---

## Before You Start

### How To Run The Project Locally

1. Open a terminal in the project folder (`UID/`).
2. Run: `python3 -m http.server 8000`
3. Open your browser and go to: `http://localhost:8000`
4. To see changes after editing: save the file, then hard refresh (`Ctrl+Shift+R`).

### Golden Rule
**Always make a backup before editing.** Copy the file you're about to change, or use `git commit` to save the current state.

## How To Change Text

### Page Titles (Browser Tab)
Open the HTML file and find the `<title>` tag (near line 7):
```html
<title>SignGloss — Real-time ISL to Gloss Translation</title>
```
Change the text between the tags.

### Hero Title & Subtitle (`index.html`)
```html
<!-- Line 32 -->
<h1 class="sg-hero-title animate-child">SignGloss</h1>

<!-- Line 33 -->
<p class="sg-hero-subtitle animate-child">Real-time Indian Sign Language to text...</p>
```

### Section Titles (Any Page)
Look for elements with class `sg-section-title`:
```html
<h2 class="sg-section-title animate-child">Pipeline</h2>
```

### Stat Numbers (`index.html`)
Change the `data-value` attribute AND the displayed text:
```html
<div class="sg-stat-number" data-value="26">0</div>
<div class="sg-stat-label">Landmarks Tracked</div>
```
The `data-value` is what the counter animates TO. The initial text (`0`) is what shows before animation.

### About Page Tile Content (`about.html`)
Find the tile overlays (starting around line 278). Each overlay has an `id` like `overlay-pipeline`:
```html
<div class="sg-tile-overlay-inner">
  <h3>Pipeline</h3>

  <!-- Edit the text here -->
</div>
```

The expanded bio is inside `<div class="sg-card-expand-content">`.

## How To Change Images

### Background Image (Most Pages)
The neuron-map background is referenced in `css/style.css` line 63:
```css
background-image: url("../assets/images/world-map-neurons.png");
```
Replace the file at `assets/images/world-map-neurons.png` or change the URL.

### Team Photos
Replace files at `assets/images/anand.jpeg` or `assets/images/vidyasree.jpeg`.
Keep the same filename, or update the `src` in `team.html`:
```html
<img src="assets/images/vidyasree.jpeg" alt="Vidyasree Jayaprasad">
```

### Learn Page Slideshow Images
Replace `assets/images/pic1.jpg` through `pic5.jpg`, or change the URLs in `learn.html` lines 81–85:
```html
<div class="sg-slide active" style="background-image: url('assets/images/pic1.jpg')"></div>
```

### About Page Background
The collage tiles use `assets/background.jpg`. Change it in `about.html` inline styles.

### To Add a New Image
1. Put the image file in `assets/images/`.
2. Reference it in HTML: `<img src="assets/images/myimage.jpg" alt="description">`.
3. Or in CSS: `background-image: url('../assets/images/myimage.jpg');`.

## How To Change Colors

### Global Colors (Easiest)
Edit CSS variables in `css/style.css` lines 9–41:

```css
/* Change main background from near-black to dark blue */
--bg-deep: #0A0A1A;

/* Change primary text from off-white to light blue */
--text-primary: #E0E8FF;

/* Change success color from green to teal */
--color-success: #2DD4BF;
```

**All elements using these variables will update automatically.**

### Button Colors
Primary button (`css/style.css` line 313):
```css
.sg-btn-primary {
  background: #F5F5F5;   /* Change this for button background */
  color: #0A0A0A;         /* Change this for button text */
}
```

### Hand Landmark Colors (`js/hand-tracking.js` lines 11–12)
```javascript
var POINT_COLOR = 'rgb(250, 44, 250)';   // Pink dots
var LINE_COLOR = 'rgb(121, 22, 76)';     // Dark pink lines
```

### Specific Element Colors
Use browser DevTools (F12) to find the element, check which CSS class styles it, then edit that class in `style.css`.

## How To Change Animations

### Disable ALL Animations
Add this at the END of `css/style.css`:
```css
* {
  animation: none !important;
  transition: none !important;
}
```

### Disable the Intro Animation Only
In `js/intro-animation.js`, change line 6 to always skip:
```javascript
// Change this:
if (sessionStorage.getItem('sg-intro-played')) {
// To this:
if (true) {
```

### Make the Intro Play Every Time
In `js/intro-animation.js`, comment out or remove line 256:
```javascript
// sessionStorage.setItem('sg-intro-played', '1');  // REMOVE THIS LINE
```

### Change Animation Speed
Find the animation or transition in CSS and change the duration:
```css
/* Slower scroll animations (0.5s → 1s) */
.animate-child {
  transition: opacity 1s ease-out, transform 1s ease-out;
}

/* Faster marquee (30s → 15s) */
.sg-marquee-track {
  animation: marquee 15s linear infinite;
}
```

### Change Scroll Animation Distance
In `css/style.css` line 141:
```css
.animate-child {
  transform: translateY(20px);   /* Change 20px to 40px for more dramatic effect */
}
```

### Change Page Transition Direction
In `js/transitions.js`:
```javascript
// Exit: currently slides LEFT (-100). Change to slide UP:
animateElements(elements, 0, 0, 1, 0, 400, ...);  // Change -100 to 0
// And add transform: translateY(-100px) instead
```

### Change Slideshow Speed (`learn.html`)
Find the `setInterval` (line 320):
```javascript
setInterval(function() { ... }, 5000);   // Change 5000 to 3000 for 3 seconds
```

## How To Add a New Page

### Step 1: Create the HTML File
Copy `demo.html` (simplest page) and save as `newpage.html`.

### Step 2: Edit the Content
```html
<title>SignGloss — Your Page Name</title>
<meta name="description" content="Description of your page">

<main>
  <section class="sg-section">
    <div class="container">
      <h1 class="sg-section-title animate-child">Your Page Title</h1>
      <p class="animate-child">Your content here...</p>
    </div>
  </section>
</main>
```

### Step 3: Add to Navigation
Edit `js/common.js`:
```javascript
const NAV_LINKS = [
  { text: 'Home', href: 'index.html' },
  // ... existing links ...
  { text: 'New Page', href: 'newpage.html' },  // ADD THIS
];
```

### Step 4: Keep Required Scripts
Make sure your page has these scripts at the bottom:
```html
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
<script src="js/common.js"></script>
<script src="js/transitions.js"></script>
<script src="js/animations.js"></script>
```

### Step 5: Test
1. Save all files.
2. Refresh your browser.
3. Check the navbar shows your new link.
4. Click the link and verify the page loads with transitions.

## How To Edit the Contact Form

### Add a New Field (`team.html`)
```html
<!-- Add after the qualification field -->
<div class="mb-3">
  <label for="contact-email" class="sg-contact-label">Email</label>
  <input type="email" class="sg-contact-input" id="contact-email" required placeholder="Your email">
  <div class="invalid-feedback">Please enter a valid email.</div>
</div>
```

Then in `js/team.js`, add validation:
```javascript
var emailInput = document.getElementById('contact-email');

emailInput.addEventListener('input', function() {
  validateField(emailInput, emailInput.value.includes('@'));
});

// In the submit handler, add:
var emailOk = validateField(emailInput, emailInput.value.includes('@'));
```

### Change Validation Rules
In `js/team.js`, find the validation conditions:
```javascript
// Change minimum name length from 2 to 3:
validateField(nameInput, nameInput.value.trim().length >= 3);

// Change minimum message length from 10 to 20:
validateField(messageInput, messageInput.value.trim().length >= 20);
```

## How To Test Changes Safely

### 1. Use Browser DevTools First
Press `F12` to open DevTools. You can:
- Edit CSS in real-time (Styles panel).
- Test color changes without modifying files.
- Inspect element sizes and positioning.
- Check the Console for JavaScript errors.

### 2. Check Responsive Design
In DevTools, click the phone/tablet icon (top left of DevTools) to test different screen sizes.

### 3. Check For Errors
Open the Console tab in DevTools. Red messages = errors. Fix them before deploying.

### 4. Test All Pages
After making changes, click through ALL pages to make sure nothing is broken:
- Home → About → Demo → Operations → Learn → Team
- Check transitions work between pages.
- Test on a small screen (resize browser window).

### 5. Test the Camera Feature
Go to Operations page and click TRANSLATE. Verify:
- Camera permission dialog appears.
- After allowing, video shows with landmarks.
- FPS counter updates.
- TERM OPS button stops everything.

## Common Mistakes To Avoid

| Mistake | What Happens | Fix |
|---|---|---|
| Missing closing tag | Page layout breaks | Count opening/closing tags |
| Typo in class name | Style doesn't apply | Copy-paste class names |
| Wrong file path | Image/script doesn't load | Check capitalization and folder |
| Editing minified CSS | Impossible to read | Only edit `style.css`, not Bootstrap |
| Forgetting `animate-child` | Element doesn't animate on scroll | Add the class |
| Opening HTML directly | CORS errors, scripts fail | Use a local server |
| Changing variable names in JS | Other code breaks | Search for all usages first |
