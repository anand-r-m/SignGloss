About SignGloss

A client-side pipeline that translates Indian Sign Language into gloss tokens using webcam input, MediaPipe, and ONNX neural networks.

Project Overview

SignGloss is a real-time Continuous Sign Language Recognition (CSLR) system that runs entirely in the browser. It captures your webcam feed, extracts 26 key hand and body landmarks using Google's MediaPipe, then feeds those features into an ONNX neural network for inference. The output is decoded using CTC (Connectionist Temporal Classification) to produce readable gloss tokens.

What the System Does

SignGloss translates Indian Sign Language (ISL) gestures into gloss tokens — standardized text labels representing individual signs. It processes continuous signing, not just isolated gestures, making it suitable for real conversation flows.

How It Works

The pipeline runs in five sequential stages, all executing client-side:

  • The webcam captures video at 30 FPS via WebRTC
  • MediaPipe extracts 26 hand and body landmarks per frame (x, y, z coordinates)
  • A Web Worker computes velocity features and buffers 64 frames into a sliding window
  • The ONNX Runtime runs neural network inference on the buffered tensor [1, 64, 156]
  • CTC greedy decoding collapses model output into readable gloss tokens

Key Components

  • MediaPipe Tasks Vision — Google's ML library for real-time hand and pose landmark detection directly in the browser
  • ONNX Runtime Web — runs the trained neural network using WebAssembly for near-native inference speed
  • Web Workers — keeps inference off the main thread so the UI stays responsive
  • CTC Decoding — Connectionist Temporal Classification handles the variable-length alignment between frames and output tokens

No data ever leaves your device. There is no server, no API, and no cloud dependency.
