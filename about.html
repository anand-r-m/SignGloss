How We Did It

SignGloss is a browser-based real-time ISL translation system. It captures webcam input, extracts landmarks with MediaPipe, runs ONNX neural network inference, and decodes output using CTC — all client-side with zero server dependency.

Pipeline

The SignGloss pipeline operates in five sequential stages, all executing client-side:

  1. Camera Capture — WebRTC captures video at 30 FPS from the user's webcam.
  2. MediaPipe Landmarks — Google's MediaPipe Tasks Vision extracts 26 hand and body landmarks per frame (x, y, z coordinates).
  3. Web Worker Processing — A dedicated Web Worker computes velocity features and buffers 64 frames into a sliding window.
  4. ONNX Inference — The ONNX Runtime runs the trained transformer on the buffered tensor with shape [1, 64, 156], producing output [1, 64, 601].
  5. CTC Decoding — Greedy CTC decoding collapses the model output into readable gloss tokens, removing blanks and repeated predictions.

Transformer Architecture

The model uses a transformer encoder architecture with positional encoding. It processes 64-frame windows of landmark data:

  • Input: 156 features per frame (26 landmarks × 3 coordinates × 2 for position + velocity)
  • Tensor shape: [1, 64, 156] → model → [1, 64, 601]
  • Output: 601 classes (600 gloss tokens + 1 blank token for CTC)
  • Positional encoding preserves temporal ordering within the 64-frame window
  • Exported as ONNX format for browser-based inference via ONNX Runtime Web (WASM backend)

Datasets Used

The model was trained on the INCLUDE ISL dataset:

  • 600 gloss classes covering common Indian Sign Language signs
  • Webcam recordings of signers performing isolated and continuous signs
  • Landmark sequences extracted via MediaPipe, normalized and augmented
  • Train/validation/test splits ensuring signer-independent evaluation

Impact

SignGloss addresses a critical communication barrier:

  • Over 1.8 million deaf and hard-of-hearing individuals in India rely on ISL
  • Runs entirely in the browser — no installation, no app store, no server costs
  • Complete privacy: no video or data ever leaves the user's device
  • Works on any modern browser with a webcam — desktop or laptop
  • Zero-cost deployment makes it accessible to schools, NGOs, and individuals

Future Scope

  • Sentence-level translation — moving beyond isolated gloss tokens to full sentence output
  • Support for additional sign languages (ASL, BSL, and others)
  • Larger and more diverse training datasets for improved accuracy
  • Mobile device support with optimized lightweight models
  • Offline capability via service workers and cached model files
  • Two-way communication: text-to-sign avatar generation
