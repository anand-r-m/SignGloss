How We Did It

SignGloss is a browser-based real-time ISL translation system. It captures webcam input, extracts landmarks with MediaPipe, runs ONNX neural network inference, and decodes output using CTC — all client-side with zero server dependency.

Our System

Pipeline
Five sequential stages, all client-side

Pipeline

The SignGloss pipeline operates in five sequential stages, all executing client-side:

  1. Camera Capture — WebRTC captures video at 30 FPS from the user's webcam.
  2. MediaPipe Landmarks — Google's MediaPipe Tasks Vision extracts 26 hand and body landmarks per frame.
  3. Web Worker Processing — A dedicated Web Worker computes velocity features and buffers 64 frames.
  4. ONNX Inference — The ONNX Runtime runs the transformer on tensor [1, 64, 156] → [1, 64, 601].
  5. CTC Decoding — Greedy CTC decoding collapses model output into readable gloss tokens.
Transformer Architecture
Encoder with positional encoding

Transformer Architecture

The model uses a transformer encoder with positional encoding, processing 64-frame windows:

  • Input: 156 features per frame (26 landmarks × 3 coords × 2)
  • Tensor shape: [1, 64, 156] → [1, 64, 601]
  • Output: 601 classes (600 gloss + 1 blank for CTC)
  • Exported as ONNX for browser inference via WASM backend
Datasets Used
INCLUDE ISL dataset

Datasets Used

Trained on the INCLUDE ISL dataset:

  • 600 gloss classes covering common ISL signs
  • Webcam recordings, landmark sequences normalized and augmented
  • Signer-independent train/validation/test splits
Impact
Bridging the communication gap

Impact

SignGloss addresses a critical communication barrier:

  • 1.8M+ deaf/hard-of-hearing in India rely on ISL
  • Browser-only — no install, no server costs
  • Complete privacy: no data leaves the device
  • Zero-cost deployment for schools, NGOs, individuals
